1. INTRODUCTION

The Pool package defines a generalized object pooling interfaces, and provides several general-purpose implementations. The following major classes are provided:

  • ObjectPool - the basic pooling interface, defining borrowObject, returnObject and several optional methods.
  • KeyedObjectPool - a pooling interface that supports pooling multiple typed objects.
  • PoolableObjectFactory - interface defining lifecycle methods for object instances used within an ObjectPool.
  • KeyedPoolableObjectFactory - interface defining lifecycle methods for object instances used within a KeyedObjectPool.
  • StackObjectPool/StackKeyedObjectPool - implementations of ObjectPool and KeyedObjectPool backed by a simple Stack of pooled instances
  • GenericObjectPool/GenericKeyedObjectPool - more complex implementations of ObjectPool and KeyedObjectPool, supporting asynchronous eviction of pooled instances, etc.

2. DEPENDENCIES

The Pool package is dependent upon the following external components for development and use:

  • Java Development Kit (Version 1.2 or later)
  • Jakarta-Commons Collections package (Version 1.0 or later)
  • JUnit Testing Framework (Version 3.7 or later) - for unit tests only, not required for deployment
